08 Learn the Fundamentals

The concepts behind the console, for anyone reading the code for the first time.

What is Encryption?

Encryption is the process of converting readable data (plaintext) into an unreadable form (ciphertext) using an algorithm and a key, so that only someone with the right key can reverse the process and read the original message.

What is Decryption?

Decryption is the reverse of encryption: taking ciphertext and, using the correct key and algorithm, recovering the original plaintext exactly as it was before encryption.

Symmetric Encryption

Symmetric encryption uses the same key to both encrypt and decrypt data. It's fast and efficient, which is why it's used for bulk data (like AES), but both parties need a secure way to share that one key in advance.

Asymmetric Encryption

Asymmetric encryption uses a mathematically linked key pair: a public key to encrypt, and a private key to decrypt. Anyone can encrypt a message for you using your public key, but only you can read it. This solves the key-sharing problem symmetric encryption has.

Caesar Cipher History

Named after Julius Caesar, who reportedly used a shift of three to protect military communications, the Caesar cipher is one of the earliest recorded encryption techniques and a foundational example in the study of cryptography.

Why the Caesar Cipher Fails

With only 25 possible keys, a computer can try every single one instantly. Worse, the cipher preserves letter-frequency patterns, so even without brute force, basic statistics reveal the key almost immediately. It offers effectively zero real-world security today.

AES Overview

The Advanced Encryption Standard is a symmetric block cipher operating on 128-bit blocks with 128, 192, or 256-bit keys. Adopted by the U.S. government in 2001, it remains the global standard for securing everything from Wi-Fi traffic to disk encryption.

RSA Overview

RSA is an asymmetric algorithm whose security rests on how computationally difficult it is to factor the product of two very large prime numbers. It's widely used for secure key exchange and digital signatures, typically alongside a faster symmetric cipher for bulk data.

When Is Each Algorithm Used?

Classical ciphers like Caesar or Vigenere are taught for their historical and educational value, not for real protection. AES secures bulk data at rest and in transit. RSA (or elliptic curve variants) handles key exchange and identity verification, often working together with AES in protocols like TLS to combine speed with secure key sharing.
